AI features

AI here is a support tool: it helps with understanding and practice, but your notes still stay central.

Notes-first retrieval

Answers start from your uploaded class material.

Optional web fallback

External context is used only when notes are missing detail.

Multi-mode outputs

One interface handles Q&A, note cleanup, and quiz generation.

Local key storage

API keys stay in browser storage on-device.

Design principle: support, not autopilot

The goal is to speed up studying without removing student thinking. AI is used to summarize, explain, and generate practice prompts, but students still control what gets accepted, edited, and reviewed. This keeps the learning process active instead of passive.

How retrieval is prioritized

When possible, answers are grounded in your own notes first. That matters for class-specific vocabulary, teacher expectations, and test wording. If your notes are incomplete, the app can pull supplemental context from web sources, but the default behavior is still notes-first.

Where AI adds the most value

The biggest gain is reducing conversion time: raw notes can quickly become summaries, flashcards, and quizzes. Another benefit is targeted explanation when a topic is confusing. Instead of manually searching multiple sources, students can ask focused questions in one place and keep the conversation tied to current study sets.

Safety and practical constraints

The app stores API keys locally in browser storage to avoid hardcoding credentials in the repository. It also keeps external retrieval optional so students can disable it when they want to stay strictly within class material. This keeps the tool flexible for different courses and teacher preferences.
